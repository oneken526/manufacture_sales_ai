処理内容
  • 検索フィルタの適用
    URLパラメータ ?q=キーワード が存在する場合、会社名・担当者名・電話番号に対して LIKE '%キーワード%'（部分一致）で絞り込む。キーワードが空の場合は全件取得。
  • ページネーション
    1ページあたり50件表示。ページ遷移時は検索キーワードを ?q=... として引き継ぎ、検索結果の2ページ目以降も正しく表示できるようにする。
  • ロールによる操作ボタンの出し分け
    同じ一覧画面でも、ログイン中のユーザーのロールに応じてボタンの表示・非表示が切り替わる。
  • 削除処理（admin のみ）
    削除ボタン押下 → 確認ダイアログ → DELETE送信。受注が存在する顧客は削除できず、顧客詳細画面にエラーメッセージを表示する。
💡 なぜそうなっているのか
  • 部分一致検索にした理由: 顧客の正式な社名を正確に覚えていなくても素早く探せるようにするため。「株式会社」「（株）」表記の揺れも吸収できる。
  • ページネーションが50件の理由: システム要件（NFR-021）で定めたパフォーマンス基準。件数が増えても画面が重くならないようにするため。
  • 削除を admin のみにした理由: 顧客データは受注・見積と紐づく重要なマスタデータのため、誤削除を防ぐために権限を絞っている。

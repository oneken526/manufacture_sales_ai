ユーザー一覧
システムに登録されているユーザーの管理を行う画面（管理者専用）

HTTPメソッド GET

項目名 説明 表示条件

状態バッジ 有効 is_active = true
無効 is_active = false 		常に表示

無効化 / 有効化ボタン 有効ユーザーには「無効化」、無効ユーザーには「有効化」ボタンを表示。確認ダイアログあり 自分自身の行には表示しない

ページネーション 1ページ50件 常に表示
処理内容
  • 一覧表示
    全ユーザーを id 昇順で取得し、50件ページネーションで表示する。
  • 有効化 / 無効化
    確認ダイアログ表示 → PATCH送信 → is_active フラグを反転させる。無効化されたユーザーはログインできなくなる。ログイン中の自分自身は無効化・有効化できない（サーバーサイドでもチェック）。
  • パスワード再設定メール送信
    POST送信後、Laravelの組み込みパスワードリセット機能（Password::sendResetLink）を利用して対象ユーザーのメールアドレスにリセットリンクを送信する。送信後は一覧画面へリダイレクトし「パスワード再設定メールを送信しました」と表示。
💡 なぜそうなっているのか
  • 削除でなく「無効化」で管理する理由: ユーザーを削除すると、そのユーザーが作成した見積・受注の「誰が操作したか」という履歴情報が失われる。無効化であればユーザーIDが残り、過去の操作ログを正確に追跡できる。
  • 自分自身を無効化できない理由: 管理者が自分のアカウントを誤って無効化すると、システムにログインできる管理者が0人になるリスクがある。これを防ぐためサーバーサイドとUIの両方でチェックしている。
  • パスワード再設定をシステム管理者が送れる理由: 社員がパスワードを忘れた際のサポートオペレーション（ヘルプデスク対応）のため。管理者が任意のタイミングでリセットメールを送ることができる。

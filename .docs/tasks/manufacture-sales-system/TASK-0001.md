## タスク概要
Laravel 13.8 / PHP 8.3を用いた本システムの開発環境を新規に構築する。プロジェクト作成、Composer依存関係の設定、Laravel Breezeによる認証scaffold生成、開発用DB（MySQL）のセットアップ、PDF生成ライブラリ（mPDF）の導入、Vite + jQuery + Bootstrap 5によるフロントエンドビルド環境の構築までを行い、以降のすべての開発タスクの土台を整備する。本タスクはコードを書く（TDD）タスクではなく、環境構築・設定作業（DIRECT）として実施する。

## 実装詳細

### 1. Laravel 13.8 / PHP 8.3 プロジェクト作成・composer.json設定 🔵
**信頼性**: 🔵 *note.md 技術スタックより*
- `composer create-project laravel/laravel`等でLaravel 13.8系のプロジェクトを新規作成する
- `composer.json`の`require`に`"php": "^8.3"`、`"laravel/framework": "^13.8"`を明記し、バージョン要件を固定する
- `composer install`が正常に完了し、`php artisan --version`でLaravel 13.8系であることを確認する
**実装ファイル**: `composer.json`, `composer.lock`, `artisan`

### 2. Laravel Breeze インストール・認証scaffold生成 🔵
**信頼性**: 🔵 *design-interview.md Q3「認証基盤の選択」回答（Laravel Breeze採用、Breeze標準のBlade構成を使用）より*
- `composer require laravel/breeze --dev`を実行し、`php artisan breeze:install`でBlade構成のscaffoldを生成する（jQueryとの共存を前提に、SPA向けスタックは選択しない）
- ログイン・新規登録・パスワードリセット・プロフィール編集の各画面が生成されることを確認する
- `npm install`で生成されたフロントエンド依存パッケージを導入する
**実装ファイル**: `routes/auth.php`, `resources/views/auth/`, `app/Http/Controllers/Auth/`, `app/Models/User.php`

### 3. MySQL開発DB設定（.env設定、データベース作成） 🔵
**信頼性**: 🔵 *architecture.md データベース構成より*
- MySQLサーバー上に開発用データベース`manufacture_sales_ai`を作成する
- `.env`の`DB_CONNECTION=mysql`、`DB_HOST`、`DB_PORT`、`DB_DATABASE`、`DB_USERNAME`、`DB_PASSWORD`を環境に合わせて設定する
- `php artisan migrate`（マイグレーションファイルが存在しない初期段階では`php artisan migrate:status`等）でDB接続を確認する
**実装ファイル**: `.env`, `config/database.php`

### 4. mPDF（mpdf/mpdf）パッケージ導入 🔵
**信頼性**: 🔵 *design-interview.md Q2「PDF生成ライブラリの選択」回答（mPDF採用、日本語帳票対応）より*
- `composer require mpdf/mpdf`でmPDFパッケージを導入する
- 日本語帳票対応のため、日本語フォント（例: ipaexg/ipaexm等）をmPDF設定に組み込む（design-interview.md「mPDF用日本語フォントの準備」を踏まえる）
- 簡易な動作確認用ルート/コントローラを一時的に用意し、Bladeテンプレートから日本語を含むPDFが正常に生成・ダウンロードできることを確認する
**実装ファイル**: `composer.json`, `config/`（mPDF設定ファイルがあれば）, 動作確認用の一時ルート（確認後に削除可）

### 5. Vite + jQuery + Bootstrap 5 のフロントエンド環境構築 🔵
**信頼性**: 🔵 *architecture.md フロントエンド構成（Bladeテンプレート + jQuery + Vite、UIライブラリ: Bootstrap 5）より*
- `npm install jquery bootstrap @popperjs/core`でjQueryとBootstrap 5を導入する
- `resources/js/bootstrap.js`または`app.js`でjQuery・Bootstrapを読み込み、`vite.config.js`にエントリポイントを設定する
- `resources/css/app.css`（またはSCSS）でBootstrap 5のスタイルを取り込む
- レイアウトBlade（`resources/views/layouts/app.blade.php`）に`@vite`ディレクティブを設定し、`npm run dev` / `npm run build`でビルドが成功することを確認する
**実装ファイル**: `resources/js/app.js`, `resources/css/app.css`, `vite.config.js`, `resources/views/layouts/app.blade.php`, `package.json`

### 6. .env.example整備、ANTHROPIC_API_KEY等Phase2向け環境変数プレースホルダー追加 🟡
**信頼性**: 🟡 *prep.mdより推測（Phase 2でClaude APIを利用する計画があるため、開発環境構築時にプレースホルダーを用意しておくのが合理的）*
- `.env.example`に本システム固有の設定項目（DB接続、メール送信設定、セッションタイムアウト等）を整理して記載する
- Phase 2のAI機能（REQ-100〜REQ-104）に向けて、`ANTHROPIC_API_KEY`、`ANTHROPIC_API_MODEL`等のプレースホルダーをコメント付きで`.env.example`に追加しておく（値は空、Phase 2実装時に設定）
- READMEまたはコメントで「Phase 2実装時に値を設定すること」を明記する
**実装ファイル**: `.env.example`

## 注意事項
- composerおよびnpmの依存パッケージのバージョンは、PHP 8.3 / Laravel 13.8との互換性を必ず確認すること
- `.env`ファイルはGit管理対象外（`.gitignore`）とし、`.env.example`のみをコミットすること。`ANTHROPIC_API_KEY`等の機密情報は`.env.example`には空のプレースホルダーのみを記載する
- mPDFの日本語フォント設定を誤ると文字化けが発生するため、動作確認時は必ず日本語を含むテンプレートでPDF生成を確認すること
- Breezeのスタック選択はBladeベース（jQueryと共存可能な構成）を選び、SPA向け（Vue/React）スタックは選択しないこと（architecture.md フロントエンド構成との整合のため）
- 本タスクで作成する動作確認用の一時ルート・コントローラは、完了条件の確認後に削除またはコメントアウトしておくこと

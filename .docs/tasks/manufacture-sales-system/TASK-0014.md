## 実装詳細

### 1. ReportServiceクラス実装（月次・年次集計、顧客別・商品別ランキング集計。SQL集約クエリ(GROUP BY)を活用し10秒以内に完了させる） 🔵
**信頼性**: 🔵 *REQ-080「月次・年次の売上集計をグラフ表示」、REQ-081「顧客別売上ランキング」、REQ-082「商品別売上ランキング」、NFR-002「レポート生成（月次集計）は10秒以内に完了」、dataflow.md「機能5」（S->>R: 集約クエリ実行（GROUP BY 顧客/商品/月））、data-types.php（SalesReportData DTO: periodType, groupBy, rows, totalAmount）より*
- `ReportService::generateSalesReport(ReportConditionData $condition): SalesReportData`: 期間種別（`monthly`/`yearly`）・集計軸（`customer`/`product`/`period`）の条件に応じて、`Repository`層の集約クエリを呼び出し、`SalesReportData` DTO（既にdata-types.phpに定義済み）として結果を返す
- `aggregateMonthly(int $year, int $month)`: 月次の売上集計（`sales_orders`/`sales_order_items`または`invoices`を起点に`SUM(quantity * unit_price)`等を`GROUP BY`で集計）を行う
- `aggregateYearly(int $year)`: 年次の売上集計を行う
- `rankByCustomer(条件)` / `rankByProduct(条件)`: 顧客別・商品別の売上ランキングを集計金額の降順でソートして返す
- 集計はDB側のSQL集約クエリ（`GROUP BY` + `SUM`/`COUNT`）に委譲し、アプリケーション層でのループ集計を避けることで、10秒以内（NFR-002）の応答時間を実現する。必要に応じてインデックス（`idx_sales_orders_customer_id`等）を活用する
**実装ファイル**: `app/Services/ReportService.php`, `app/Repositories/Contracts/SalesReportRepositoryInterface.php`, `app/Repositories/Eloquent/SalesReportRepository.php`

### 2. ReportController実装（レポート画面表示、CSVエクスポート） 🔵
**信頼性**: 🔵 *api-endpoints.md（GET /reports/sales, GET /reports/sales/export）、dataflow.md「機能5」（V->>C: GET /reports/sales?period=monthly&group=customer / V->>C: GET /reports/sales/export?...）より*
- `index(Request $request)`: `GET /reports/sales` - クエリパラメータ（`period`: monthly/yearly, `group`: customer/product/period）に応じて`ReportService::generateSalesReport()`を呼び出し、グラフ・ランキングを画面表示する。アクセス可能ロールはsales, accounting, admin
- `export(Request $request)`: `GET /reports/sales/export` - 同条件でレポートデータを取得し、`ReportService::exportCsv()`を介してCSVストリームをダウンロードレスポンスとして返す。アクセス可能ロールはsales, accounting, admin
- ロールごとのアクセス制御は`EnsureUserHasRole`ミドルウェアを利用する
**実装ファイル**: `app/Http/Controllers/ReportController.php`, `routes/web.php`

### 3. グラフ表示の実装（jQuery + Chart.js等のJSライブラリ連携） 🟡
**信頼性**: 🟡 *REQ-080「システムは月次・年次の売上集計をグラフで表示できなければならない」から推測。具体的なグラフライブラリ（Chart.js等）はarchitecture.mdに明記がなく未確定のため、jQueryとの親和性が高く軽量なChart.jsを想定して記述*
- `ReportController::index()`から渡された`SalesReportData`の`rows`（`label`, `amount`の配列）をJSON形式でビューに渡し、jQuery経由でChart.js（棒グラフ・折れ線グラフ）を初期化して描画する
- 月次推移は折れ線グラフ、顧客別・商品別ランキングは横棒グラフでの表示を想定する
- グラフの再描画は期間・集計軸の切り替え時にAJAXでデータを再取得し、jQueryでDOM更新後にChart.jsインスタンスを更新する
**実装ファイル**: `resources/views/reports/sales.blade.php`, `resources/js/reports-chart.js`

### 4. CSVエクスポート機能（レポートデータをCSV形式でダウンロード） 🔵
**信頼性**: 🔵 *REQ-083「システムはレポートデータをCSV形式でエクスポートできなければならない」、dataflow.md「機能5」（S-->>C: CSVストリーム / C-->>V: CSVファイルダウンロード、詳細ステップ4: 大量データでもメモリ効率を確保）より*
- `ReportService::exportCsv(条件): StreamedResponse`: `Illuminate\Support\Facades\Response::streamDownload()`等を用いて、集計結果をCSV形式でストリーミング出力する
- ヘッダー行（集計軸名・金額等の列名）と各集計行をCSV形式で書き出し、大量データでもメモリ効率を確保する（dataflow.md詳細ステップ4）
- 文字コードはExcel等での文字化けを防ぐためBOM付きUTF-8またはShift-JISへの変換を考慮する
**実装ファイル**: `app/Services/ReportService.php`, `app/Http/Controllers/ReportController.php`

### 5. sales/accounting/admin役割によるアクセス制御 🟡
**信頼性**: 🟡 *REQ-084「管理職・経理担当者と営業担当者はレポートを閲覧できなければならない」、api-endpoints.md（GET /reports/sales, GET /reports/sales/export: sales, accounting, admin）から推測*
- `ReportController`の`index()`・`export()`アクションには`EnsureUserHasRole:sales,accounting,admin`ミドルウェアを適用する
- warehouse等、上記以外のロールがアクセスした場合は403エラー（権限エラー画面）を返す
**実装ファイル**: `routes/web.php`, `app/Http/Controllers/ReportController.php`

## 注意事項
- 集計処理は必ずDB側のSQL集約クエリ（`GROUP BY`, `SUM`, `COUNT`）に委譲し、アプリケーション層での全件取得・ループ集計を行わないこと（NFR-002の10秒以内要件を満たすため）
- 集計対象期間が長期にわたる場合や対象データ量が多い場合に備え、必要なインデックス（`idx_sales_orders_customer_id`, `idx_sales_orders_status`等）の活用状況を確認し、必要であれば追加のインデックス検討を設計レビューで提起すること
- CSVエクスポートは`StreamedResponse`によるストリーミング出力とし、大量データでもメモリ枯渇が起きない実装とすること（dataflow.md詳細ステップ4）
- グラフ描画ライブラリ（Chart.js等）の選定はarchitecture.mdに明記されていないため、実装時にプロジェクトの技術選定方針を確認のうえ確定すること（🟡項目）
- ロールベースアクセス制御は`EnsureUserHasRole`ミドルウェアに委譲し、Controller内でのロール判定ロジックの重複を避けること（REQ-084）
- 集計対象データ（`sales_orders`, `invoices`等）の状態（キャンセル済み・返品済み等）の扱いについては、業務要件に照らして適切な除外条件をクエリに含めること

## タスク分割の方針 🔵

**信頼性**: 🔵 *ユーザーヒアリングより*

- タスクは「画面単位」ではなく**「機能単位（縦方向に貫通）」**で分割。1機能＝DB設計〜Repository〜Service〜Controller〜Bladeビューまでを1タスクとして完結させる
- 実装順序は機能を縦に貫通させる方式（基盤構築 → マスタ管理 → 受注フロー → 請求/レポート → 横断・統合）を採用
- 単体テストは主要業務ロジック（在庫整合性・ステータス遷移・採番・権限制御等）を重点的にカバーする方針

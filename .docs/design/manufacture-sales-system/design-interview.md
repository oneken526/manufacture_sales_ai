# 製造業向け販売管理システム 設計ヒアリング記録

**作成日**: 2026-06-07
**ヒアリング実施**: Step4 既存情報ベースの差分ヒアリング

## ヒアリング目的

新規プロジェクトのため、技術設計の方針（フロントエンド構成・PDF生成・認証基盤・アーキテクチャパターン・採番ルール・在庫設計）を確定するヒアリングを実施。

---

## 質問と回答

### Q1: フロントエンドのUI構築方法

**カテゴリ**: 技術選択
**背景**: Blade + Vite構成だが、UIインタラクションの実装方法を確定する必要がある

**回答**: jQuery

**信頼性への影響**:
- アーキテクチャの「フロントエンド」項目が🔴→🔵に向上
- Blade + jQuery + Vite（アセットビルド）構成で確定
- AJAX通信は内部APIエンドポイント（Laravel標準のWeb/APIルート）経由で実施

---

### Q2: PDF生成ライブラリの選択

**カテゴリ**: 技術選択
**背景**: 請求書・納品書・見積書の日本語帳票PDF出力（REQ-052, REQ-061, REQ-032）に必要

**回答**: mPDF（推奨案を採用）

**信頼性への影響**:
- REQ-032, REQ-052, REQ-061の実現方法が🟡→🔵に向上
- 日本語フォント（IPAフォント等）の同梱が必要（prep.mdに追記）

---

### Q3: 認証基盤の選択

**カテゴリ**: アーキテクチャ
**背景**: REQ-001（ログイン）、REQ-005（パスワードリセット）の実装基盤を確定

**回答**: Laravel Breeze（推奨案を採用）

**信頼性への影響**:
- REQ-001, REQ-005の実現方法が🟡→🔵に向上
- Breeze標準のBlade構成を使用（jQueryと共存可能）

---

### Q4: バックエンドアーキテクチャパターン

**カテゴリ**: アーキテクチャ
**背景**: 受注フロー（見積→受注→出荷→請求）は状態遷移と在庫整合性が複雑なため、設計パターンの確定が必要

**回答**: リポジトリパターン

**信頼性への影響**:
- アーキテクチャ全体の構成が🔴→🔵に向上
- Repository + Service層構成で実装方針を確定（テスト容易性・DB切替への対応も考慮）

---

### Q5: 採番ルール

**カテゴリ**: データモデル
**背景**: REQ-060（請求書発行）・見積書（REQ-030）の番号体系を確定する必要がある

**回答**: 年度+連番（例: INV-2026-0001）

**信頼性への影響**:
- 請求書・見積書・受注番号のフォーマットが🟡→🔵に向上
- DBスキーマに採番用のテーブルまたはロジックを設計

---

### Q6: 在庫の引き当てと実減算の区別

**カテゴリ**: データモデル
**背景**: REQ-040（受注時引き当て）とREQ-051（出荷時実減算）をデータ上どう表現するか確定が必要

**回答**: 二つのカラム（stock_quantity / reserved_quantity）

**信頼性への影響**:
- products テーブルの設計が🟡→🔵に向上
- 利用可能在庫 = stock_quantity - reserved_quantity という計算式が確定

---

## ヒアリング結果サマリー

### 確認できた事項
- フロントエンド: Blade + jQuery + Vite（SPAフレームワークなし）
- PDF生成: mPDF（日本語帳票対応）
- 認証: Laravel Breeze
- アーキテクチャ: Repository + Service層パターン
- 採番ルール: 年度+連番形式
- 在庫設計: stock_quantity（実在庫）と reserved_quantity（引当中）の2カラム

### 設計方針の決定事項
- ルーティング: Web routes（画面）+ 内部APIルート（jQuery AJAX用）の併用
- AI機能（Phase 2）: Claude API用に別途APIエンドポイントを設計
- 在庫の利用可能数 = stock_quantity - reserved_quantity で算出し、受注時にバリデーション

### 残課題
- mPDF用日本語フォントの準備（prep.mdに追加）
- 全銀協フォーマットCSVの列構成の詳細確認（実装時に詳細化）

### 信頼性レベル分布

**ヒアリング前**:
- 🔵 青信号: 0件
- 🟡 黄信号: 6件
- 🔴 赤信号: 6件

**ヒアリング後**:
- 🔵 青信号: 12件（+12）
- 🟡 黄信号: 0件（-6）
- 🔴 赤信号: 0件（-6）

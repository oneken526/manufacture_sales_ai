# 製造業向け販売管理システム コンテキストノート

## プロジェクト概要

製造業の販売管理業務をweb上で一元管理するシステム。受注から出荷・請求までの一連のフローをデジタル化し、売上分析・AI予測機能も提供する。

## 技術スタック

| 項目 | 内容 |
|------|------|
| フレームワーク | Laravel 13.8 |
| 言語 | PHP 8.3 |
| DB（開発） | SQLite |
| DB（本番候補） | MySQL / PostgreSQL |
| フロントエンド | Blade + Vite |
| テスト | PHPUnit 12 |
| AI機能（Phase 2） | Claude API（Anthropic） |

## 開発フェーズ

### Phase 1（初期リリース）
- 顧客・製品マスタ管理
- 受注管理（見積→受注→出荷指示→出荷完了→請求）
- 請求書・納品書PDF出力
- 在庫数管理（シンプル）
- 売上レポート（月次・顧客別・商品別・CSVエクスポート）
- ユーザー認証・権限管理（4役割）

### Phase 2（AI機能追加）
- 売上・需要予測
- 顧客購買パターン分析
- チャットAI Q&A（自然言語でデータ照会）
- 在庫最適化提案

## ユーザー役割

| 役割 | 主な権限 |
|------|---------|
| システム管理者 | 全機能・ユーザー管理・マスタ設定 |
| 営業担当者 | 顧客・見積・受注・請求書閲覧 |
| 在庫・出荷担当者 | 在庫・出荷操作（請求不可） |
| 管理職・経理担当 | 全閲覧・レポート・請求管理 |

## 受注ステータスフロー

```
[見積] → [受注確定] → [出荷指示] → [出荷完了] → [請求発行] → [入金確認]
                ↓
           [キャンセル]
                        ↓
                   [返品処理]
```

## 注意事項

- 在庫管理はシンプル（数量のみ）— 仕入れ先管理・発注機能はスコープ外
- PDF生成ライブラリの選定が必要（DomPDF / mPDF 候補）
- 銀行振込データ形式（全銀協フォーマット）の取り込みが必要
- AI機能はPhase 2のため、Phase 1実装時はAPI連携の準備のみ

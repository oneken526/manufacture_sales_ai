## REQ-001: ユーザーログイン 🔵

### Given
- ログイン画面が表示されている
- 有効なユーザーアカウントが存在する

### When
- ユーザーがメールアドレスとパスワードを入力してログインする

### Then
- 認証成功時、役割に応じたダッシュボードにリダイレクトされる
- 認証失敗時、エラーメッセージ「メールアドレスまたはパスワードが正しくありません」が表示される

### テストケース

#### 正常系
- [ ] **TC-001-01**: 有効な認証情報でログイン → ダッシュボードへリダイレクト 🔵
- [ ] **TC-001-02**: システム管理者でログイン → 管理者メニューが表示される 🔵
- [ ] **TC-001-03**: 営業担当者でログイン → 営業メニューのみ表示される 🔵

#### 異常系
- [ ] **TC-001-E01**: 誤ったパスワードでログイン → エラーメッセージ表示 🔵
- [ ] **TC-001-E02**: 存在しないメールアドレス → エラーメッセージ表示 🔵
- [ ] **TC-001-E03**: 無効化されたアカウント → ログイン拒否メッセージ 🟡

## REQ-003: 権限によるアクセス制御 🔵

### Given
- 在庫・出荷担当者としてログインしている

### When
- 請求書発行画面のURLに直接アクセスする

### Then
- 403エラーページ（アクセス権限なし）が表示される
- 元の画面（ダッシュボード）にリダイレクトされる

### テストケース
- [ ] **TC-003-01**: 在庫担当者が請求書URL直打ち → 403リダイレクト 🔵
- [ ] **TC-003-02**: 営業担当者がユーザー管理URL直打ち → 403リダイレクト 🟡

## REQ-010: 顧客登録 🔵

### Given
- 営業担当者以上の権限でログインしている

### When
- 顧客情報（会社名・担当者名・住所・電話・メール・与信枠）を入力して保存する

### Then
- 顧客が登録され一覧に表示される
- 会社名は必須項目でバリデーションエラーの場合はフォーム上に表示される

### テストケース
- [ ] **TC-010-01**: 必須項目すべて入力して保存 → 登録成功 🔵
- [ ] **TC-010-02**: 会社名を空白で保存 → バリデーションエラー 🔵
- [ ] **TC-010-E01**: 同一会社名の重複登録 → 警告（許容または拒否は要確認） 🟡

## REQ-040〜043: 受注確定・在庫引き当て 🔵

### Given
- 見積書が作成済みである
- 対象製品の在庫数が受注数量以上ある

### When
- 「受注確定」ボタンをクリックする

### Then
- ステータスが「受注」に変わる
- 在庫数が引き当て数として確保される（実際の出荷時に減算）

### テストケース
- [ ] **TC-040-01**: 在庫十分な状態で受注確定 → 正常受注 🔵
- [ ] **TC-040-02**: 在庫不足の状態で受注確定 → 警告表示、処理中止 🔵
- [ ] **TC-043-01**: 受注キャンセル → 引き当て在庫が戻る 🔵

## REQ-051〜052: 出荷完了登録・納品書PDF 🔵

## REQ-060〜064: 請求書発行・入金管理 🔵

### Given
- 管理職・経理担当者としてログインしている
- 出荷完了済みの受注が存在する

### When
- 請求書を発行してPDFをダウンロードする

### Then
- 請求書番号が自動採番されて発行される
- PDFに請求先・請求金額・振込先が正しく記載される
- 入金ステータスが「未入金」で作成される

### テストケース
- [ ] **TC-060-01**: 請求書発行 → PDF出力・未入金ステータス設定 🔵
- [ ] **TC-060-02**: 同一受注に二重発行 → 警告・2回目は拒否 🟡
- [ ] **TC-062-01**: 入金ステータスを「入金済み」に更新 → 正常更新 🔵
- [ ] **TC-063-01**: 全銀CSVインポート → 照合成功件数・失敗件数を表示 🔵
- [ ] **TC-063-02**: 照合成功した請求書 → 自動的に「入金済み」更新 🔵
- [ ] **TC-063-E01**: 存在しない振込データ → スキップして未照合として報告 🔵

## REQ-070〜072: 在庫管理 🔵

### Given
- 製品マスタが登録されている

### When
- 受注確定・出荷完了・返品・手動調整が行われる

### Then
- 在庫数が正しく更新される
- 変動履歴（日時・理由・数量・操作者）が記録される

### テストケース
- [ ] **TC-070-01**: 在庫一覧表示 → 全製品の現在庫数が表示される 🔵
- [ ] **TC-072-01**: 出荷完了後 → 在庫変動履歴に記録が追加される 🟡
- [ ] **TC-072-02**: 手動調整後 → 変動履歴に「手動調整」として記録 🟡

## REQ-080〜083: 売上レポート 🔵

### Given
- 管理職・経理担当者または営業担当者としてログインしている
- 受注・請求データが存在する

### When
- 売上レポート画面を表示する

### Then
- 月次・年次の売上集計グラフが表示される
- 顧客別・商品別売上ランキングが表示される
- CSVエクスポートボタンからデータを取得できる

### テストケース
- [ ] **TC-080-01**: 月次売上グラフ表示 → 当月の集計が正しく表示される 🔵
- [ ] **TC-081-01**: 顧客別ランキング → 売上降順で表示される 🔵
- [ ] **TC-082-01**: 商品別ランキング → 売上降順で表示される 🔵
- [ ] **TC-083-01**: CSVエクスポート → UTF-8 BOMつきCSVがダウンロードされる 🟡

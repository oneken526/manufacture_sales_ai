## 必須（実装開始前に完了が必要）

- [ ] **PDFライブラリの選定・インストール決定** 🔵 *請求書・納品書PDF要件（REQ-060, REQ-052）より*
  - 候補: `barryvdh/laravel-dompdf`（軽量）または `mpdf/mpdf`（日本語対応◎）
  - 日本語フォント対応の観点からmPDFを推奨
  - 関連要件: REQ-052, REQ-061

- [ ] **全銀協フォーマットCSVのサンプルファイル取得** 🔵 *振込データインポート要件（REQ-063）より*
  - 取引銀行のネットバンキングからダウンロード手順を確認
  - 銀行によってフォーマットに差異がある場合があるため確認必須
  - 関連要件: REQ-063

---

## 推奨（実装中に用意できればOK）

- [ ] **本番データベースの選定** 🟡 *インフラ要件（NFR-030）より*
  - 候補: MySQL 8.0+ または PostgreSQL 15+
  - 本番環境のホスティング先（レンタルサーバー・VPS・クラウド）と合わせて決定
  - 必要になるフェーズ: Phase 1本番リリース前
  - 関連要件: NFR-030

- [ ] **Claude APIキーの取得（Phase 2用）** 🔵 *AI機能要件（REQ-100〜104）より*
  - Anthropic Console（console.anthropic.com）でAPIキーを発行
  - `.env`に`ANTHROPIC_API_KEY=`として設定
  - 必要になるフェーズ: Phase 2開始前
  - 関連要件: REQ-104

- [ ] **メール送信設定** 🟡 *パスワードリセット（REQ-005）・請求書送付から推測*
  - SMTPサーバー情報（またはMailgunなどのメール配信サービス）
  - `.env`の`MAIL_*`設定値が必要
  - 必要になるフェーズ: Phase 1（パスワードリセット実装時）

---

## 確認事項（判断が必要）

- [ ] **在庫数0時の受注許容可否の最終確認** 🟡 *在庫管理フロー（EDGE-010）から推測*
  - 現在の設計：在庫0の場合は受注不可（警告表示）
  - 受注生産型の製品については例外を設けるか要判断
  - 関連要件: EDGE-010

- [ ] **請求書番号の採番ルールの確認** 🟡 *請求書発行（REQ-060）から推測*
  - 例：`INV-2026-0001`（年度+連番）など
  - 既存の社内ルールがある場合は合わせる必要がある
  - 関連要件: REQ-060

- [ ] **顧客重複チェックの方針確認** 🟡 *顧客登録（REQ-010）から推測*
  - 同一会社名の登録を許容するか拒否するか
  - 関連要件: REQ-010
